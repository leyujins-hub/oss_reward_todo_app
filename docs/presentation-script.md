# 발표 대본 초안

## 1. 배경 및 동기

저희 팀은 대학생들이 과제, 시험, 팀플 일정을 자주 놓치거나 미루는 문제에 주목했습니다. 기존 To-Do List는 할 일을 단순히 나열하는 방식이기 때문에 어떤 일이 더 급한지, 어떤 일을 계속 미루고 있는지 직관적으로 알기 어렵습니다.

그래서 저희는 울산대학교 학생들을 대상으로 한 Adaptive To-Do List 앱을 기획했습니다.

## 2. Adaptiveness 설명

저희 앱의 Adaptiveness는 사용자의 작업 상태에 따라 UI와 추천 방식이 변화하는 것입니다.

마감일, 중요도, 미완료 횟수, 완료 패턴을 기준으로 작업의 우선순위를 바꾸고, 위험도를 색상으로 표시합니다.

예를 들어 마감일이 가까운 과제는 상단에 배치되고, 중요도가 높은 과제는 더 강한 색상으로 표시됩니다. 또한 여러 번 미룬 과제는 경고 문구를 통해 사용자가 바로 인식할 수 있도록 했습니다.

## 3. Concept

저희 앱은 단순히 과제를 체크하는 앱이 아니라, 과제를 완료할수록 캐릭터와 캠퍼스가 성장하는 보상형 To-Do List입니다.

사용자는 앱을 처음 열었을 때 성장하기 전의 울산대 캐릭터 화면을 보게 되고, 과제를 수행할수록 캠퍼스가 점점 발전하는 구조입니다.

## 4. 주요 기능

주요 기능은 할 일 추가, 수정, 삭제, 완료 체크, 마감일 설정, 중요도 설정, 카테고리 분류입니다.

추가로 오늘의 추천 작업, 미완료 항목 경고, 캐릭터 성장, 카테고리별 통계 기능을 포함했습니다.

## 5. Git 협업

저희 팀은 GitHub 기반 협업 방식을 사용했습니다.

main 브랜치는 최종 제출용, develop 브랜치는 개발 통합용으로 사용했고, 각 기능별로 feature 브랜치를 나누어 작업했습니다.

커밋 메시지는 feat, fix, style, docs로 구분하여 작성했습니다.

작업이 끝난 기능은 Pull Request를 통해 develop 브랜치에 병합했고, 최종 제출 버전에는 v1.0.0 태그를 추가하여 버전 관리가 가능하도록 했습니다.

## 6. Future Work

향후에는 GPS 기반 학교 도착 알림, 학교 사이트 바로가기, 공부 시간 측정 기능, 사진 인증 또는 퀴즈 기반 완료 인증 기능을 추가할 수 있습니다.
