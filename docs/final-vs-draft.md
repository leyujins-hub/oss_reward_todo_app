# 프롬프트 결과 초안과 최종 구현본 비교

## 1. 비교 목적

본 프로젝트에서는 생성형 AI와 Figma AI를 활용하여 초기 화면 초안과 기능 아이디어를 도출했다. 그러나 생성형 AI가 제안한 결과물을 그대로 사용한 것이 아니라, 프로젝트 목적, 발표 기간, 구현 가능성, UI 통일성, 실제 배포 안정성을 고려하여 최종 구현본을 재설계했다.

따라서 `design/draft` 폴더에는 팀원별 프롬프트 입력 결과로 나온 초안 화면을 정리했고, `design/final` 폴더에는 최종 사이트에서 실제로 확인할 수 있는 구현 화면을 정리했다.

---

## 2. 안혜원 초안과 최종 구현본 비교

### 초안 방향

안혜원 팀원은 다기능 Todo List와 기린 캐릭터 성장 시스템을 중심으로 프롬프트를 입력했다. 초안에서는 과제 추가, 마감일, 중요도, 카테고리, 상태 변경 기능과 함께, 과제를 완료할 때마다 EXP가 증가하고 기린 캐릭터가 Lv.1부터 Lv.5까지 성장하는 구조가 제안되었다.

### 최종 반영

최종 구현본에서는 Todo List의 기본 기능과 기린 캐릭터 성장 카드가 반영되었다. 사용자는 할 일을 추가하고, 중요도와 마감일을 설정할 수 있으며, 과제를 완료하면 EXP가 증가한다. 상단에는 기린 캐릭터, 현재 레벨, 누적 EXP, 다음 레벨까지 필요한 EXP, 오늘의 진행도가 표시된다.

### 차이점

초안에서는 5단계 캐릭터 외형 진화가 제안되었지만, 최종 구현본에서는 레벨과 EXP 구조를 우선 구현했다. 캐릭터 외형이 단계별로 완전히 바뀌는 기능은 이미지 리소스와 상태 전환 로직이 추가로 필요했기 때문에 Future Work로 분리했다.

### 차이가 발생한 이유

발표 기간 안에 안정적으로 구현하기 위해, 시각적 진화 전체보다 보상 시스템의 핵심인 EXP, 레벨, 진행도 표시를 우선했다. 즉, 캐릭터 성장의 기반 구조를 먼저 구현하고, 외형 변화는 추후 확장 가능한 구조로 남겼다.

## 3. 김민지 초안과 최종 구현본 비교

### 초안 방향

김민지 팀원은 오늘의 명언 카드와 통계 페이지를 중심으로 프롬프트를 입력했다. 초안에서는 하루에 한 번 날짜 기반으로 다른 명언이 표시되고, 1주일, 1개월, 1년 단위로 사용자의 활동을 그래프와 성장 통계로 확인하는 기능이 제안되었다.

### 최종 반영

최종 구현본에서는 통계 페이지가 핵심 기능으로 반영되었다. 전체 퀘스트 수, 완료 수, 진행 중 수, 완료율이 자동으로 계산되며, 주간·월간·연간 활동 그래프가 제공된다. 또한 성장 통계, 업적, 연속 수행 기록, 총 집중 시간, 완료된 과제 평균 집중 시간이 표시된다.

### 차이점

초안에 포함된 오늘의 명언 기능은 최종 시연의 핵심 기능에서는 비중이 축소되었다. 반면 통계 페이지는 최종 구현본에 더 강하게 반영되었고, 집중 시간 데이터와 연결되어 성장 분석 기능까지 확장되었다.

### 차이가 발생한 이유

명언 기능은 사용자 동기부여 측면에서 의미가 있지만, 사용자의 실제 행동 데이터와 직접 연결되는 기능은 아니었다. 최종 발표에서는 과제 완료, 집중 시간, EXP, 성장 통계처럼 사용자의 행동 결과를 보여주는 기능이 더 핵심이라고 판단했다. 그래서 명언은 보조 동기부여 아이디어로 두고, 통계와 성장 분석을 우선 구현했다.

## 4. 김도연 초안과 최종 구현본 비교

### 초안 방향

김도연 팀원은 과제별 집중 시간 타이머와 빠른 링크 기능을 중심으로 프롬프트를 입력했다. 초안에서는 집중 시간이 0분인 과제는 시작 필요, 일정 시간 이상 집중한 과제는 진행 중 또는 거의 완료 가능으로 표시하고, 학교 관련 사이트를 빠른 링크 카드로 제공하는 기능이 제안되었다.

### 최종 반영

최종 구현본에서는 과제별 집중 시간 타이머와 빠른 링크 기능이 반영되었다. 사용자는 각 과제 카드에서 타이머를 시작, 일시정지, 종료할 수 있고, 기록된 집중 시간은 통계 페이지의 총 집중 시간과 평균 집중 시간에 반영된다. 또한 Classum, UWINS, 학사/장학 공지, 도서관 좌석 예약으로 이동할 수 있는 빠른 링크 카드가 구현되었다.

### 차이점

초안에서는 링크 커스텀, 드래그 앤 드롭 순서 변경, 개인 링크 추가 기능까지 제안되었지만, 최종 구현본에서는 기본 학교 링크 4개를 우선 제공하는 방식으로 정리되었다. 집중 시간 상태 배지 역시 모든 색상 규칙을 완전히 세분화하기보다는 실제 시연 가능한 타이머 기록 기능 중심으로 구현되었다.

### 차이가 발생한 이유

링크 커스텀과 드래그 앤 드롭 기능은 추가적인 상태 관리와 localStorage 저장 구조가 필요했다. 발표 기간과 배포 안정성을 고려하여, 최종본에서는 학교 사이트 접근성을 높이는 핵심 기능인 빠른 링크 4개를 우선 구현했다. 집중 시간 기능도 상태 배지의 복잡한 분기보다 실제 시간 기록과 통계 연결을 우선했다.

## 5. 최종 구현본으로 통합한 기준

최종 구현본은 팀원들의 초안을 모두 그대로 합친 결과가 아니라, 다음 기준에 따라 기능을 선별하고 통합한 결과이다.

1. 보상형 Todo List라는 핵심 컨셉과 직접 연결되는가
2. 실제 사이트에서 안정적으로 시연 가능한가
3. 사용자의 행동 데이터와 연결되는가
4. 발표 시간 안에 설명 가능한가
5. UI 전체 분위기와 일관성이 있는가

이 기준에 따라 최종 구현본은 Todo List 기본 기능, 기린 캐릭터 성장, EXP 시스템, 중요도 기반 카드 강조, 집중 시간 타이머, 통계 페이지, 빠른 링크 기능을 중심으로 구성되었다.

---

## 6. 발표 시 답변 요약

초안과 최종본이 다른 이유는 기능이 누락된 것이 아니라, 발표 기간 안에 안정적으로 구현 가능한 핵심 기능 중심으로 통합했기 때문이다. 생성형 AI가 제안한 화면은 아이디어 도출과 UI 방향 설정에 활용했고, 최종 구현본은 프로젝트 목적과 실제 사용 흐름에 맞게 재설계했다.
